## 大切にしていること

### 分析を、より良い意思決定につなげる

私は、技術を選ぶ前に「何を解くべきか」「なぜその選択が必要か」を明らかにすることを大切にしています。
優れたアーキテクチャは、先進的な技術を組み合わせるだけでは生まれません。
事業やプロダクトの目的、ドメインの構造、データの所有と流れ、求める品質、運用するチーム、そして時間やコストの制約までを捉え、それぞれの関係とトレードオフを踏まえて設計する必要があると考えています。

私にとって分析は、机上で正解を探すためのものではなく、複雑な状況を構造化し、チームが納得して次の一歩を選ぶための営みです。
曖昧な要求を問いに変え、境界や依存関係を可視化し、「作らない」という選択肢も含めて比較する。
判断の根拠を言葉にしてステークホルダーと共有し、不確実性が残る部分は評価基準を定めたPoCで確かめる。
こうした過程を通じて、短期的な成果と中長期的な持続可能性を両立させます。

ビジネス、ドメイン、データ、アーキテクチャ、組織を横断して考え、抽象的な構想を実装・運用可能な形へ落とし込めることが私の強みです。
データ基盤や新規事業の立ち上げ、プロダクト開発、組織横断の技術戦略で培った経験を生かし、難しい問題に対して判断できる材料と共通言語をつくり、チームが自律的に前へ進める状態をつくります。
